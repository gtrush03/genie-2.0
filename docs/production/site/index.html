01 — Packaging

Desktop App

From dev-mode Node server to a downloadable .dmg. Electron wins because Genie needs a controllable Chromium — and Electron IS a controllable Chromium.

Why Electron (not Tauri)

Tauri's WebView can't do Chrome DevTools Protocol. You'd ship Tauri + Playwright Chromium sidecar = a worse Electron. Electron's bundled Chromium doubles as both the app UI and the agent browser.

FrameworkBundleCDP SupportVerdict
Electron~150 MBNativeV1 Winner
Tauri 2.0~3 MBNoneWrong fit
Wails~10 MBNoneNo advantage
Native Swift~5 MBManualMac only

Bundle Contents

ComponentSize

Claude Agent SDK + node_modules~15 MB
Playwright MCP~5 MB
Genie source + skills + config~1 MB
Total .dmg (compressed)~80-90 MB

Comparable to VS Code (~100 MB). Smaller than Slack (~170 MB).

Claude Integration: Agent SDK vs CLI

The Agent SDK (@anthropic-ai/claude-agent-sdk) replaces spawning claude -p as a subprocess. Direct library call, same capabilities, no CLI binary needed.

Auto-Update Strategy

Two-tier system: Bug fixes and new skills deploy in minutes (hot update). Electron/SDK changes go through a full release cycle.

App Shell (electron-updater)

  • Code-signed Mac + Windows

Hot Updates (skills + prompts)

  • CDN-hosted skills-manifest.json
  • On-launch version check
  • Download only changed files

Preferences

Learned from behavior. Beer brand, tip amount, design aesthetic, posting tone. Confidence-scored with 60-day decay.

Corrections

Highest priority. "I said Modelo, not Corona" creates a permanent override that never decays. Carries negative signal.

Routines

"My usual" = auto-detected after 3 similar wishes. Explicit shortcuts like "post about this" map to concrete specs.

Profile Schema (full)

Identity & Services

  • JellyJelly username, Telegram chat ID
  • 12+ connected services with health status
  • Addresses (default + home)
  • Payment methods (masked)

Learned Data

  • Food prefs: brands, stores, tip amounts
  • Posting: tone, platform, avoid-list
  • Design: aesthetic tokens, colors, fonts
  • Outreach: sign-off style, tone

03 — Safety

Edge Cases & Risk Register

Known risks, mitigations, and the safety architecture for an autonomous agent that controls real accounts.

RiskSeverityLikelihoodImpactMitigation
Wrong account action
Agent posts to wrong social account		 Critical Medium Public embarrassment, account damage Service health check before action. Account identity verification step in system prompt.
Financial over-spend
Agent orders $500 of food or creates expensive invoices		 Critical Low Unintended charges $25 per-wish budget cap. Uber Eats skill has explicit "do not place order" confirmation step. Stripe in test mode by default.
Session expiry mid-wish
Login cookie expires during browser automation		 High Medium Wish fails silently Service health check on startup + every 6 hours. Telegram alert on expired sessions. Wish retry with fresh check.
Transcript misinterpretation
Deepgram mishears wish, agent executes wrong intent		 High Medium Wrong action taken Correction loop ("Genie, not that, I meant..."). All actions logged with undo hints where possible.
Chrome OOM / crash
Chromium runs out of memory under heavy tab use		 Medium Medium Wish fails, needs restart Docker restart policy. 3 GB memory limit per container. Health check ping on CDP endpoint.
API key exposure
Anthropic key leaked in logs or deployed site		 Critical Low Financial liability, account compromise Keys in .env only, never in profile JSON. Electron safeStorage for keychain. Log redaction middleware.
Rate limiting
Anthropic API or LinkedIn throttles requests		 Medium High Wishes queued, delayed Exponential backoff. Queue overflow to Supabase. Per-user rate limit in Redis.
Duplicate wish execution
Same clip processed twice		 Medium Low Duplicate posts/orders Clip ID dedup set in Redis. Idempotency checks in dispatcher.

Safety Architecture Layers

1
Budget Cap
$25/wish, 200 turns max, 60-min timeout
2
Service Check
Verify logged-in status before action
3
Killswitch
Global Redis flag stops all wishes instantly
4
Audit Log
Every action logged with undo hints

06 — Experience

User Onboarding

From download to first wish in under 5 minutes. The app bootstraps itself.

1

Download & Install

User downloads .dmg from genie.app or Homebrew Cask (brew install --cask genie). Drag to Applications. ~80 MB download.

2

API Key Setup

First launch asks for Anthropic API key (or Claude Max subscription). Stored in system keychain via Electron's safeStorage API. Optional: Telegram bot token for reporting.

3

Browser Login

Genie opens a Chrome window with tabs for X, LinkedIn, Gmail, Uber Eats, Vercel, GitHub, and more. User logs into each (check "Keep me signed in"). Sessions persist across app restarts.

4

JellyJelly Connection

User enters their JellyJelly username. Genie starts polling the firehose for clips containing the keyword "genie." Status shows green when connected.

5

First Wish

User records a JellyJelly video: "Genie, build me a landing page for my AI startup." Genie detects the keyword, executes the wish, deploys a live site, and sends a Telegram report. The user watches their browser move on its own.

The Flywheel Effect

flowchart LR
    A["Post more on\nJellyJelly"] --> B["Genie learns\nyour preferences"]

First wish: generic landing page. Fifth wish: knows your brand, audience, network. Tenth wish: proactively suggests strategy before you ask.
